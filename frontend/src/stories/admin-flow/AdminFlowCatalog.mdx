## 設計意図

- `/admin` に管理責務を集約し、参加者機能を混在させない。
- 直接アクセス拒否時は「管理画面本体を描画しない」ことを UI 仕様として明示する。
- 権限判定中に操作を無効化し、誤操作やチラつき表示を防ぐ。

## ストーリー確認観点

- `AdminAuthorized`: 管理機能のみ表示され、参加者機能が排除されているか
- `AdminPermissionChecking`: 判定中に管理操作を無効化できているか
- `AdminForbidden`: 拒否理由と復帰導線のみ表示し、管理UIを表示しないか
